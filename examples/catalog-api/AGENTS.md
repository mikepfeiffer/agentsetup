# catalog-api — Agent Instructions

Inherits the root `AGENTS.md`. The rules below are **catalog-api-only** and are
intentionally stricter than the repo defaults. This is a self-contained
reference: it carries both its own instructions and its own deterministic hooks
(`.agent-hooks/` + the `.claude/` and `.codex/` adapters). Launch the agent from
this directory so those hooks are active.

## Stack
ASP.NET Core 8 minimal API + EF Core (SQLite). The guardrail tooling is portable
Node (see `.agent-hooks/`); the app is .NET.

## Commands
- **Build:** `dotnet build CatalogApi.sln`
- **Test:** `dotnet test CatalogApi.sln` (xUnit; integration tests use an
  in-memory SQLite database).
- **Run:** `dotnet run --project src/CatalogApi` (applies migrations, seeds, serves).
- **Add a migration:** `dotnet tool restore` then
  `dotnet dotnet-ef migrations add <Name> --project src/CatalogApi`.
- **Hook tests:** `npm run test:hooks`.

## Module rules
- **IMPORTANT: Never hand-edit files under `Migrations/`.** They are generated by
  `dotnet ef`. To change the schema, change the model and **add a new migration**.
  The hook blocks edits to `*ModelSnapshot.cs` and `*.Designer.cs`.
- **IMPORTANT: Money is `decimal`, never `double`/`float`.** `decimal` is exact
  base-10; binary floats introduce rounding error. Format to a string only at the
  display edge.
- **Public HTTP contract.** Return RFC 7807 `ProblemDetails` for errors (use
  `Results.Problem` / `Results.ValidationProblem`); never leak exceptions or
  stack traces. Don't change a response shape or status code without a versioned,
  backward-compatible path.
- **Validate at the edge.** Every endpoint validates input before touching the
  database (`ProductValidation`); never trust the caller.
- **Never commit secrets.** Only `appsettings.json` (placeholders / non-secret
  config) is committed. Real secrets live in `dotnet user-secrets` or env vars;
  the hook blocks `appsettings.Production.json`, `*.pfx`, `secrets.json`, etc.
- **Never edit build output (`bin/`, `obj/`).** It is generated by the SDK; the
  hook blocks it too.
