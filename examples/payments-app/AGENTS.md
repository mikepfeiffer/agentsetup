# payments-app — Agent Instructions

Inherits the root `AGENTS.md`. The rules below are **payments-app-only** and are
intentionally stricter than the repo defaults — this module is the reference for
how nested, module-specific guidance layers on top of the shared root.

## Commands
- **Build:** `npm run build` (compiles `src/` + `test/` to `dist/`).
- **Test:** `npm test` (compiles, then runs `node --test` on `dist/test`).
- **Type-check only:** `npm run typecheck`.

## Module rules
- **IMPORTANT: Money is integer minor units (cents). Never use floats.** All
  amounts are whole numbers of cents; rendering to a decimal happens only at the
  display edge (`formatCents`). A float anywhere in a money path is a bug.
- **IMPORTANT: Charges must be idempotent, keyed by `requestId`.** Retries are
  expected and must never double-charge. Reusing a `requestId` with a *different*
  amount is a conflict and must throw, not silently overwrite.
- **Validate at the edge.** Every public entry point (`charge`, the CLI) checks
  its input before doing work — never trust the caller.
- **Keep `money.ts` pure.** Its functions are side-effect free and safe to reuse
  anywhere; treat their signatures as a stable API.
- **Never commit secrets.** Real config lives in a local `.env` (git-ignored and
  hook-blocked). Only `.env.example` — with placeholders — is committed.
- **Never edit `dist/`.** It is generated by `tsc`; change `src/` instead. (The
  shared hook also blocks edits to `dist/` and `node_modules/`.)
